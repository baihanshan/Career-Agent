# Sprint 2 输出质量人工 QA Checklist

## 使用方式

使用 `backend/tests/fixtures/fix_quality_profile.md` 与 `fix_quality_jd.txt` 发起一次完整分析。每次评审记录版本、模型、日期和评审人；各项按 1～5 分评分，低于 4 分需附失败示例。相同输入和配置应保留结果，以便版本间对比。

## 硬性检查

- [ ] 页面文本不存在 `evidence_id`、requirement ID、chunk ID 或空内部 ID。
- [ ] 学历等静态资格不生成“如何满足要求”类问题。
- [ ] Python/算法与多模态问题包含具体场景、约束和技术权衡。
- [ ] 简历深挖问题只用项目名或短摘要定位，不粘贴完整经历。
- [ ] 示例答案围绕问题重新组织，不大段复述简历。
- [ ] Python、CV/ML、NLP、RAG、多模态经历未被误判为缺失。
- [ ] “计算机视觉、NLP 或多模态”满足任一分支即可视为覆盖。
- [ ] 列表编号、日期和 `DeepLabV3+` 不触发未经支持的数字风险。

## 判定规则

- 硬性检查必须全部通过。
- 六项评分均不得低于 4 分，平均分不得低于 4.2 分。
- 未通过时保留原始问题、答案或风险文本，并关联对应失败维度。
- 修复后使用同一输入、模型和温度复测，记录前后评分差异。
